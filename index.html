01

Service Overview

아몬즈 고객은 가격·혜택에 민감하고(여성 91%, 25–34세 MZ 중심), 여러 상품을 비교하며 구매해요. 그래서 ‘비교 구매’ 흐름이 전환을 좌우하는 핵심 접점이었어요.

02

Background

구매 전환을 높이려면 퍼널 곳곳을 봐야 했는데, 그중 장바구니 진입률이 유독 낮았어요(13.45% · App 기준). 이 지점을 개선 대상으로 잡았어요.

03

User Definition

장바구니를 ‘구매 직전 상품을 비교하는 공간’으로 정의하고, 더 많이 담고 더 자주 들어오게 만드는 걸 목표로 삼았어요. (OKR: 전 분기 대비 판매 수 증가)

04

Problem

진입률이 낮은 건 ‘담는 행동에 접근하기 어렵기 때문’이라는 가설을 세웠고, 데이터로 두 가지 문제를 확인했어요.

문제점 01
17%

활성 사용자 중 담기를 쓴 비율 → 담기 버튼 노출 범위 확대

문제점 02
38%

담은 뒤 다시 들어오지 않은 비율 → 담은 다음의 이동 경로 제공

05

UX Problem & Solution

사용자를 ‘장바구니를 어려워하는 그룹’과 ‘이미 잘 쓰는 그룹’으로 나누고, 각 그룹의 진입률을 높일 가설을 하나씩 세웠어요.

06

A/B Test

Test 01 · 장바구니 이동 경로실패

담기 직후 이동 경로(스낵바) 제공

담은 직후 ‘장바구니로 가기’ 경로(스낵바)를 띄워 다음 행동을 알려주면 진입률이 오를지 검증했어요. 효과는 미미했어요(진입 +0.57).

Test 02 · 장바구니 담기 버튼성공

버튼 노출 범위를 넓히고 텍스트로 직관성 강화

좋아요 페이지의 상품 카드에서 바로 담을 수 있도록 텍스트 버튼을 노출했어요. 담는 행동의 인지 비용이 낮아지면서 담기율·구매 전환율이 함께 올랐어요.

07

Result

두 번의 실험 끝에, 버튼 노출을 넓힌 쪽이 더 큰 전환 효과를 만들었고 하락하던 담기 추세도 완화했어요.

Why Global?

K-Beauty 시장은 빠르게 커지는데, 글로벌 크리에이터가 쓰기 좋은 탐색 구조는 아직 없었어요.

하지만 글로벌 크리에이터와 K-뷰티 브랜드가 직접 만나 거래·탐색할 구조가 비어 있었어요 — ‘구조적 공백’이었죠.

02

Structural Difference

국내용은 크리에이터·MD가 수동으로 추천하는 구조였어요. 이걸 글로벌 크리에이터 기준으로 타깃·핵심 기능·추천 방식·데이터 연결까지 처음부터 다시 정의했어요. 핵심은 자체 데이터 LSI(Leferi Selects Index)의 4축(Brand·Product·Creator·Content)을 추천에 직접 연결한 점이에요.

03

Service Structure

크리에이터(탐색 → 선택 → 체험 → 콘텐츠), 브랜드(파트너 센터), 내부 어드민(백오피스) — 세 주체의 흐름을 하나의 구조로 연결했어요.

04

데이터 기반 탐색 구조

크리에이터가 ‘무엇을 고를지’ 판단할 기준을 홈에 모았어요. LSI Top 5 Products, 크리에이터 리뷰 콘텐츠, 급상승 상품(LSI Rising)을 데이터로 보여줬어요.

05

시딩 기반 구매 구조

시딩 티켓·포인트로 상품을 받고, 구매한 뒤 리뷰 작성까지 자연스럽게 이어지도록 흐름을 설계했어요.

06

신뢰 검증 구조

YouTube·Instagram·TikTok 채널 인증을 거친 크리에이터만 가입하도록 온보딩을 설계해, 신뢰를 먼저 확보했어요.

01

Background

이미 데이터 툴이 있었는데도, 비슷한 데이터 요청이 계속 반복되고 있었어요.

한 달 슬랙 메시지 중
52%

요청 → 전달
2~3일

의사결정 속도의 병목으로 이어짐

02

Research

2년치 슬랙 요청을 분석해 반복 유형을 뽑았어요. 브랜드 840건, 상품 361건, 크리에이터 312건, 콘텐츠 148건. 공통으로 필요한 기능은 기간 선택(169건), 유·무가 구분(123건), 로우 데이터 추출(99건) 순이었어요.

03

IA Design

반복되는 요청을 LSI의 4개 엔티티(Brand·Product·Creator·Content) IA로 재정렬했어요. ‘전체 랭킹 → 검색·선택 → 상세 → 연관 데이터’라는 공통 탐색 패턴을 만들어, 4개 엔티티를 모두 같은 방식으로 쓰게 했어요.

04

Brand Analysis

가장 많던 브랜드 랭킹 요청부터 직접 탐색하도록 만들었어요. 특정 브랜드를 골라야만 보이던 구조를, 전체 랭킹과 TOP 10의 6개월 추이를 한눈에 보는 화면으로 바꿨어요.

05

Global Filter

요청하고 기다려야 했던 조건 변경(기간·카테고리·브랜드)을 모든 화면에 적용되는 전역 필터로 바꿨어요. 기간 선택·브랜드 비교 모달도 함께 설계했고요.

User Survey

75명 설문에서 경조사비 경험자의 73%가 ‘기록해 본 적 없다’고 답했어요. 이유는 ‘귀찮아서(34.5%)’와 ‘기록이 어려워서(12.1%)’ — 이 둘을 핵심 문제로 정의했어요.

05

Features
